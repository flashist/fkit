# Make `/fkit-task-cancelled` flip the moved brief's own `## Status` header

## ID

Sprint 2

## Status
✅ Done

## Context

**`/fkit-task-cancelled` has the same gap as `/fkit-task-done`** (see task 34): it flips the sprint-plan
Status cell to `⛔ Cancelled (YYYY-MM-DD) — <reason>` and moves the brief into
`ai-agents/tasks/cancelled/`, but **never updates the moved brief's own `## Status` field.** A
cancelled brief therefore keeps reading `🔲 Backlog` (or whatever it last held) internally, while the
board reads `⛔ Cancelled`.

**Same drift, same class as task 22 and task 34** — a mover updating one record of a task's state and
leaving another stale. It's split from task 34 because the two skills are separate files, each
independently shippable, and the cancelled marker carries extra structure (a **mandatory date and
reason**) that its brief header must reproduce faithfully — a distinct correctness requirement worth
its own verification.

**Scope note — no scaffold copy.** As with task 34, the only canonical source is
`claude/skills/fkit-task-cancelled/SKILL.md`; the skills are not under `claude/scaffold/`, and the
`.claude/skills/` copy is gitignored and regenerated by init. Do not edit the copy.

## What to build

Extend `/fkit-task-cancelled`'s procedure so that **after moving the brief into `cancelled/`, it also
sets the moved brief's own `## Status` field** to the terminal cancelled marker.

Requirements:

- **Set the brief's `## Status` to the full canonical cancelled marker** —
  `⛔ Cancelled (YYYY-MM-DD) — <reason>` — using the **same date and reason** the skill writes into the
  sprint-plan board. The brief header and the board must agree, including the reason text (the
  vocabulary makes the reason mandatory, so a bare `⛔ Cancelled` in the brief is nonconformant).
- **Idempotent.** Re-running on a brief already carrying the cancelled marker leaves the `## Status`
  section byte-identical — no second marker, no re-stamped date, no appended reason.
- **A brief with no `## Status` section is left untouched and flagged** in the report, never invented.
- **Only the brief just moved** is touched — no historical sweep.
- **Still no commit.**

Update the skill's report contract so a run states the brief's own header was set (or flagged absent).

## Verification steps

- **Run `/fkit-task-cancelled` on a backlog brief with a reason and confirm the moved file's
  `## Status` reads the full `⛔ Cancelled (YYYY-MM-DD) — <reason>`** — matching the board cell exactly,
  date and reason included.
- **Idempotency:** run it again against the same (now-in-`cancelled/`) brief; the `## Status` section
  is byte-identical — one marker, the original date preserved, no duplicated reason.
- **Reason/date fidelity:** the date and reason in the brief header equal those written into the
  sprint-plan Status cell — not a truncated or reworded copy.
- **Missing-section case:** on a brief with no `## Status` heading, the body is unchanged and the run
  flags the absent section.
- **Board still correct:** the sprint-plan cell (and any body `**Status:**` line) still flips to
  `⛔ Cancelled …` as before.
- **A subsequent `/fkit-status` shows no board-vs-brief drift** and no nonconformance flag (reason
  present in both places) for a task cancelled by the updated skill.
