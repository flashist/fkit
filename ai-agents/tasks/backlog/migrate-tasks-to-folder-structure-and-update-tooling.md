## Context

The structural change itself: every task becomes a folder, and every fkit tool that constructs or
parses a task path learns the new shape.

**This task is atomic by necessity.** The 13 product source files locate a brief by following a path
of the form `ai-agents/tasks/<board>/<slug>.md`. The moment the files move, every one of those tools
is looking at a path that no longer resolves — `dashboard.sh` stops finding briefs, both movers stop
finding rows, `/fkit-status` reports drift on everything. **Moving the files and teaching the tools
cannot ship separately**, so they are one task.

The **sprint-board hrefs are included here for the same reason**: `dashboard.sh` reads the Brief
column's href to locate each brief. Leave those pointing at the old paths and the board breaks
entirely. Repairing the *other* ~110 documentation references is task 77, which is genuinely separable
because nothing executes them.

**The 13 files to update:** `claude/fkit-claude-init.sh`, `claude/agents/fkit-coder.md`,
`claude/agents/fkit-producer.md`, and the skills `fkit-initiate-project`, `fkit-plan-task`,
`fkit-status/SKILL.md`, `fkit-status/dashboard.sh`, `fkit-task-brief`, `fkit-task-done`,
`fkit-task-cancelled`, `fkit-task-ship-loop`, `fkit-wiki-ingest`, `fkit-wiki-sync`.

**⚠️ Edit the canonical sources in `claude/` — never the `.claude/` copies**, which are gitignored and
regenerated by `claude/fkit-claude-init.sh .`.

**⚠️ Task 64 collides with this task.** Both rewrite `fkit-task-done` and `fkit-task-cancelled`.
Task 74's design states the ordering; follow it rather than deciding here.

**⚠️ ADR-027 dual-home parity:** `claude/scaffold/ai-agents/tasks/` must land the same structure, or
the parity convention breaks on the next check.

## What to build

- **The migration** — all 89 briefs into `ai-agents/tasks/<board>/<ID>-<slug>/<brief-filename>` per
  task 74's design, using `git mv` so history survives.
- **The 13 product source files** updated to construct and parse the new layout.
- **`dashboard.sh`** taught the new brief-location contract — it is the single most load-bearing
  consumer, and its drift detection reads the brief's own `## Status` from inside the folder.
- **Both task movers** updated to move a **folder**, not a file, between boards — and to keep repairing
  hrefs as they already do.
- **The sprint-board hrefs** in `ai-agents/sprints/` (~94 refs) re-pointed to the new paths.
- **The scaffold** (`claude/scaffold/ai-agents/tasks/`) landed with the same structure per ADR-027.

## Verification steps

- `ls ai-agents/tasks/{backlog,done,cancelled}/*.md` returns **nothing** — no flat brief files remain.
- Every one of the 89 task folders contains a brief at the filename task 74 specified; count the
  folders and confirm it is 89.
- `bash .claude/skills/fkit-status/dashboard.sh ai-agents/sprints/sprint-2.md` exits 0, emits
  `⟦fkit-dashboard v1⟧`, reports the **same** roll-up counts as before the migration, and emits
  **no** `missing-brief`, `relocated`, or `brief-missing-status` drift records.
- Every href in `ai-agents/sprints/*.md` and `ai-agents/sprints/done/*.md` resolves to a real path —
  checked mechanically, not by eye.
- `/fkit-task-done` and `/fkit-task-cancelled` each moved end-to-end on a scratch task: the **folder**
  moves, the board row flips, the brief's own `## Status` flips, hrefs re-point.
- `git log --follow` on a migrated brief still shows its pre-migration history — proof `git mv` was
  used rather than delete-and-create.
- The launcher-contract suite still passes.
- `claude/scaffold/ai-agents/tasks/` matches the live structure (ADR-027 parity).
