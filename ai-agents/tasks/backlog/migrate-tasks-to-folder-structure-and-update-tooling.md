## Context

The structural change itself: every task becomes a folder, every task artifact moves inside it, and
every fkit tool that constructs or parses a task path learns the new shape.

**This task is atomic by necessity.** The product source files locate a brief by following a path
of the form `ai-agents/tasks/<board>/<slug>.md`. The moment the files move, every one of those tools
is looking at a path that no longer resolves — `dashboard.sh` stops finding briefs, both movers stop
finding rows, `/fkit-status` reports drift on everything. **Moving the files and teaching the tools
cannot ship separately**, so they are one task.

The **sprint-board hrefs are included here for the same reason**: `dashboard.sh` reads the Brief
column's href to locate each brief. Leave those pointing at the old paths and the board breaks
entirely. Repairing the *other* ~110 documentation references is task 77, which is genuinely separable
because nothing executes them.

**The file set is larger than this brief originally scoped.** It said 13 files under `claude/`; task
74's design measured **21 under `claude/`, plus 4 test files and 2 `reviews/README.md` copies`**. **Take
the measured set from the design spec §8 and re-derive it yourself** — do not work from a list in a
brief:

```sh
grep -rl 'tasks/' claude/
```

**⚠️ Edit the canonical sources in `claude/` — never the `.claude/` copies**, which are gitignored and
regenerated by `claude/fkit-claude-init.sh .`.

**⚠️ Three top-level directories are absorbed, not one.** Owner ruled 2026-07-19 (design spec §4.3)
that `ai-agents/plans/`, `ai-agents/worklogs/` **and** `ai-agents/reviews/` all fold into the task
folder as `plan.md`, `worklog.md`, `review.md`. All three are keyed by the same slug today; absorbing
only `reviews/` would leave two directories slug-keyed while briefs are ID-keyed. **All three
directories cease to exist.**

**⚠️ The scaffold is UNCHANGED — do not touch it.** *(Corrected 2026-07-19: this brief previously said
the scaffold "must land the same structure". That instruction was impossible to follow — obeying it
literally would copy **this project's** task folders into a consuming project's scaffold, and the
condition it asked you to verify cannot be satisfied by a deliberately-empty scaffold.)*

Per design spec §4.1 / ADR-029 Decision 9: `claude/scaffold/ai-agents/tasks/{backlog,done,cancelled}/`
ship three **empty** board directories with `.gitkeep` files, and they **stay exactly as they are.** A
task folder always contains at least a brief, so it never needs a `.gitkeep` of its own, and a fresh
project starts with no tasks. **ADR-027 dual-home parity holds for free**, and
`test/converge-contract.test.js` needs no change.

## What to build

- **The migration** — every brief into `ai-agents/tasks/<board>/<ID>-<slug>/<brief-filename>` per
  task 74's design, using `git mv` so history survives. **Derive the brief count** with
  `ls ai-agents/tasks/{backlog,done,cancelled}/*.md | wc -l`; do not trust a figure written in a brief
  or on the sprint board — every such figure recorded so far has gone stale before the task ran.
- **`ai-agents/plans/`, `ai-agents/worklogs/` and `ai-agents/reviews/` absorbed** into their task
  folders as `plan.md`, `worklog.md`, `review.md`, by `git mv`, and the three directories removed.
  Derive their contents; do not trust a count.
- **The two sprint-scoped review ledgers moved to `ai-agents/sprints/reviews/`** —
  `sprint2-scaffold-launcher-hardening.md` and `sprint2-shared-instructions-delivery.md` are keyed to a
  sprint theme, not a task, so they have no task folder to fold into. Design spec §5.2b / ADR-029
  Decision 7. **`ai-agents/reviews/` still disappears.** *(Ruled 2026-07-19 — this brief previously told
  you to stop and ask the owner here. Do not stop: the question is answered.)*
- **`ai-agents/reviews/README.md`** — its ledger-key rule relocated per design spec §4.3, in **both**
  homes (live and `claude/scaffold/`), since `reviews/` ceases to exist.
- **The measured product source files** updated to construct and parse the new layout.
- **`dashboard.sh`** taught the new brief-location contract — it is the single most load-bearing
  consumer, and its drift detection reads the brief's own `## Status` from inside the folder.
- **Both task movers** updated to move a **folder**, not a file, between boards — and to keep repairing
  hrefs as they already do.
- **The sprint-board hrefs** in `ai-agents/sprints/` (~94 refs — derive it) re-pointed to the new paths.
- **Nothing in `claude/scaffold/ai-agents/tasks/` changed** — see the warning above.
