## Finding 1 (methodology — not a defect): the brief's `clean $HOME` is impossible for the interactive half

The brief prescribes verifying from a clean `$HOME`. That works for the **install** half and **breaks
the interactive half**: faking `$HOME` makes Claude Code look for credentials in the temp dir, and the
session cannot authenticate. Observed live during this run.

**Resolution used:**
- Install / scaffold / self-update → clean `HOME=$(mktemp -d)`. The brief's intent holds.
- Interactive steps → **real `$HOME`** (for Claude auth), isolated instead via `FKIT_SHARE` /
  `FKIT_BIN`. The install is still pristine; only auth comes from the real home.

The brief should be amended to say so, or the next person repeats the dead end.
