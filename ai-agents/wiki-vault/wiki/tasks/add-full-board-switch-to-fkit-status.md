# Add a full-board switch to `/fkit-status`

**Source**: `ai-agents/tasks/done/add-full-board-switch-to-fkit-status.md`
**Status**: done
**Sprint/Tag**: Sprint 2 — priority 38

## Goal
**`/fkit-status` had no way to force the full board on a repeat call in the same session.** Step 5 says a repeat status reports the **delta** — a good default that keeps a second call terse. But once the owner had run it once, there was **no slash-command path back to the complete dashboard**; even a named sprint (`/fkit-status Sprint 2`) counted as a repeat and deltaed.

## Key Changes
- **A reserved `full` keyword** (aliases `all` / `board`) forcing the complete seven-beat status with the **full step-4 dashboard — every task row**.
- **The keyword is reserved and must not be parsed as a sprint name.** `/fkit-status full` → active sprint, full board; `/fkit-status Sprint 2 full` → Sprint 2, full board. **`full` alone is never treated as a sprint name** — *the one thing to get right is disambiguation: it must be recognized **before** the parser tries to match a sprint name, or `/fkit-status full` breaks.*
- **The delta-on-repeat default stays** — the keyword is an **escape hatch, not a replacement**.
- **Skill-instruction text only.** *"There is no separate parser to write; the skill is prose the producer agent follows."*

**Why a switch and not a new skill** — the reasoning is about drift, and it is the recurring shape of fkit's design calls: a separate `fkit-sprint-dashboard` skill would either **duplicate step 4's dashboard spec** — *the most intricate spec the producer owns, and now two copies that drift* — or be a thin wrapper. **It would also cost a new role-locked skill registration**: `skills_for_role()` plus **the two hand-maintained mirrors** — *the three-way drift surface the test suite exists to guard.* **A switch on the existing skill avoids all of it.**

## Outcome
**Done.** Risk low — no product/runtime code, no launcher change, **no new skill registration, no mirror to update**.

**No scaffold copy exists**: the skills live only in `claude/skills/`, so the single canonical source is `claude/skills/fkit-status/SKILL.md`. **The `.claude/` copy is gitignored and regenerated by init — do not edit it.**
